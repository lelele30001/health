
          

# AI 健康助手实现流程与架构

## 1. 实现架构

### 1.1 技术栈
- **前端**：Vue 3 + Element Plus + Axios
- **后端**：Spring Boot + Java
- **AI 模型**：智谱 AI GLM-4-Flash（永久免费模型）
- **数据库**：MySQL（存储用户问题）

### 1.2 核心组件
- **前端组件**：AIChat.vue（聊天界面）
- **后端服务**：AIChatService（AI 交互逻辑）
- **控制器**：AIChatController（API 接口）
- **数据存储**：ai_question 表（存储用户问题）

## 2. 实现流程

### 2.1 前端流程
1. **用户输入**：用户在聊天界面输入健康相关问题
2. **保存问题**：前端调用 `/project/ai/save-question` 接口保存问题到数据库
3. **构建请求**：构建包含用户消息和历史记录的请求数据
4. **发送请求**：调用 `/project/ai/chat` 接口获取 AI 回复
5. **显示回复**：将 AI 回复显示在聊天界面

### 2.2 后端流程
1. **接收请求**：AIChatController 接收前端请求
2. **系统提示词**：添加系统提示词 "你是一个专业的健康饮食顾问，请用专业友好的语气回答用户问题。"
3. **构建消息**：整合系统提示词、历史记录和用户消息
4. **调用 AI API**：调用智谱 AI GLM-4-Flash 模型 API
5. **处理响应**：解析 AI 回复，处理错误情况
6. **返回结果**：将 AI 回复返回给前端

### 2.3 数据流向
```
用户输入 → 前端保存问题 → 数据库存储 → 前端发送聊天请求 → 后端处理 → 智谱 AI API → 后端解析 → 前端显示
```

## 3. 核心功能

### 3.1 系统提示词
- **提示词内容**："你是一个专业的健康饮食顾问，请用专业友好的语气回答用户问题。"
- **作用**：引导 AI 以专业的健康饮食顾问身份回答问题，确保回复贴合健康领域

### 3.2 错误处理
- **网络错误**：处理 HTTP 429 等错误，实现重试机制
- **API 错误**：处理智谱 AI API 返回的错误信息
- **系统错误**：处理系统内部异常

### 3.3 数据保存
- **保存内容**：用户 ID、问题内容、时间戳
- **保存时机**：用户发送消息时自动保存
- **保存目的**：用于推荐算法分析，捕捉用户兴趣

### 3.4 历史记录处理
- **过滤错误信息**：排除错误提示和系统消息
- **保留有效对话**：只传递有效的用户和 AI 对话

## 4. 接口设计

### 4.1 保存问题接口
- **路径**：`/project/ai/save-question`
- **方法**：POST
- **参数**：`userId`（用户 ID）、`question`（问题内容）
- **返回**：保存结果

### 4.2 聊天接口
- **路径**：`/project/ai/chat`
- **方法**：POST
- **参数**：`message`（用户消息）、`history`（历史记录）
- **返回**：AI 回复内容

## 5. 优势与特点

1. **专业定位**：通过系统提示词确保 AI 以健康饮食顾问身份回答
2. **数据驱动**：用户问题自动保存，为推荐算法提供数据支持
3. **错误处理**：完善的错误处理机制，提高系统稳定性
4. **用户体验**：流畅的聊天界面，支持快速提问和实时回复
5. **免费模型**：使用智谱 AI GLM-4-Flash 永久免费模型，降低成本

## 7. 总结

AI 健康助手通过整合前端交互、后端处理和智谱 AI 模型，实现了一个专业的健康饮食顾问功能。系统通过系统提示词引导 AI 以专业身份回答问题，同时自动保存用户问题用于推荐算法分析，形成了数据驱动的个性化推荐闭环。这种设计不仅提升了用户体验，也为系统的智能推荐能力提供了数据支持。
