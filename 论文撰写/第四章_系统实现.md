# 第四章 系统实现

智慧膳食分享与个性化推荐系统基于 Spring Boot 和 Vue 3 框架进行开发，系统实现过程中采用前后端分离的开发模式，前端负责页面展示和用户交互，后端负责业务逻辑处理和数据存储。通过需求分析、系统设计、编码实现，最终完成了预期的功能，包括用户认证、个人中心、食谱管理、社区互动、AI 助手和系统管理等核心功能模块。本章将以部分界面效果、图表可视化和关键模块实现为例，详细介绍系统的开发过程。

## 4.1 部分效果展示

### 4.1.1 系统功能概述

本系统面向健身人群，提供智慧膳食分享与个性化推荐服务，主要功能包括用户管理、个人中心、食谱浏览与推荐、社区互动、AI 智能助手和管理后台等模块。系统分为前台用户端和后台管理端两部分，前台面向普通用户提供食谱推荐和社区交流功能，后台面向管理员提供数据管理和统计功能。

### 4.1.2 主要界面展示

系统主要界面包括以下几个方面：

**(1) 用户登录与注册界面**

用户登录界面采用简洁直观的设计，用户输入用户名和密码后，系统验证用户身份并生成 Token 用于后续请求的身份认证。注册界面要求用户填写基本信息，包括用户名、密码、确认密码等，系统对输入进行验证后保存用户信息。

**(2) 系统主界面**

用户登录后进入系统主界面，主界面顶部显示导航栏，包括首页、食谱、社区、AI 助手等功能入口。中间区域展示个性化推荐食谱列表，根据用户的健身目标和历史行为进行推荐。界面设计采用 Element Plus 组件库，风格统一、美观大方。

**(3) 个人中心界面**

个人中心模块提供用户信息管理、健康档案管理、发布内容管理和收藏管理等功能。用户可以查看和编辑个人信息，设置健身目标，管理健康档案，查看自己发布的帖子和食谱，管理收藏记录。

**(4) 食谱浏览界面**

食谱列表页面支持分页展示和食谱搜索，用户可以查看食谱的名称、封面图片、健身目标、营养标签等信息。食谱详情页面展示完整的食谱内容，包括食材列表、烹饪步骤、营养成分等信息。

**(5) 社区交流界面**

社区模块提供帖子发布、评论、点赞、收藏和消息通知等功能。用户可以发布膳食经验分享，查看其他用户的帖子，进行互动交流。系统对用户发布的内容进行审核管理，确保社区内容的安全性和质量。

**(6) 管理后台界面**

管理员后台包括用户管理、食谱管理、食材管理、帖子管理、评论管理和统计报表等功能模块。管理员可以查看用户列表、搜索用户、查看食谱列表、搜索食谱、审核帖子和评论、删除违规帖子和评论、查看系统统计数据等。

## 4.2 图表可视化实现

### 4.2.1 ECharts 图表应用

系统统计报表模块采用 ECharts 技术实现数据可视化展示，主要涉及的统计图表包括用户性别分布饼状图、用户年龄分布柱状图、用户健身目标分布柱状图等。ECharts 是基于 Canvas 的开源可视化图表库，提供了丰富的图表类型和良好的交互体验。

### 4.2.2 图表实现过程

系统前端在项目中引入 ECharts 依赖后，创建图表容器并初始化图表实例。统计页面加载时，前端向后端请求统计数据，后端查询数据库获取用户统计数据并返回给前端。前端接收到数据后，配置图表选项并渲染图表，将数据以直观的可视化方式展示给用户。

管理员可以通过统计报表页面直观地了解系统的用户分布情况，包括不同性别、年龄段和健身目标的用户数量，为系统优化和运营决策提供数据支持。

## 4.3 关键模块实现

### 4.3.1 项目结构与开发环境

**(1) 项目命名与开发工具**

根据智慧膳食系统的特点，项目命名为 fitness-diet-system，后端项目名为 server_code，前端项目名为 manage_code（管理端）和 client_code（用户端）。后端开发工具选用 IntelliJ IDEA，前端开发工具选用 Visual Studio Code。数据库选用 MySQL 8.0，使用 Navicat for MySQL 进行数据库管理。

**(2) 前台软件结构及主要配置**

前端系统核心代码位于 src 目录中，主要包括以下文件夹：

- `/src/assets` - 静态资源文件，包括图片、样式表等
- `/src/components` - 通用组件，包括编辑器、图片上传、验证滑块等
- `/src/views` - 页面视图，按功能模块划分为 foods（食材）、forum（社区）、personal（个人中心）等目录
- `/src/router` - 前端路由配置，定义页面跳转路径
- `/src/store` - 状态管理，管理用户信息和系统状态
- `/src/utils` - 工具函数，包括 HTTP 请求封装、数据处理等

前端项目采用 package.json 文件管理依赖模块，主要依赖包括 Vue 3、Element Plus、Vue Router、Vuex、Axios、ECharts 等。vite.config.js 配置了开发环境和生产环境的接口地址，并配置代理解决跨域问题。

**(3) 后端软件结构及主要配置**

后端系统核心代码位于 src/main/java/com/cl 目录中，主要包括以下包：

- `/controller` - 控制器层，处理 HTTP 请求
- `/service` - 服务层，实现业务逻辑
- `/dao` - 数据访问层，操作数据库
- `/entity` - 实体类，定义数据结构
- `/config` - 配置类，系统配置
- `/utils` - 工具类，提供通用功能
- `/annotation` - 注解类，定义自定义注解

后端项目采用 pom.xml 文件管理依赖，主要依赖包括 Spring Boot、MyBatis-Plus、MySQL Connector、Apache Shiro、Spring Security 等。application.yml 配置了数据库连接信息、端口号等系统参数。

### 4.3.2 用户认证模块

**(1) 功能描述**

用户认证模块实现用户注册、登录和密码修改等功能。系统支持普通用户注册和登录，管理员通过专门的管理员入口登录后台系统。

**(2) 实现过程**

用户注册时，前端页面收集用户输入的用户名、密码等信息，通过 Axios 发送注册请求到后端。后端接收请求后，验证输入信息的合法性，将用户信息保存到数据库。密码使用 MD5 算法加密存储，确保用户密码安全。

用户登录时，前端发送用户名和密码到后端，后端验证用户身份后生成 Token 并返回给前端。前端保存 Token 到本地存储，后续请求携带 Token 进行身份认证。后端通过 Token 验证用户身份，确保请求的合法性和安全性。

密码修改时，用户需要输入原密码和新密码，后端验证原密码正确后，将新密码加密保存到数据库。

**(3) 核心代码**

用户认证控制器 YonghuController 处理注册、登录、密码修改等请求，调用 YonghuService 进行业务处理，数据访问层通过 YonghuDao 操作数据库。

### 4.3.3 个人中心模块

**(1) 功能描述**

个人中心模块实现用户信息管理、健康档案管理、发布内容管理和收藏管理等功能，帮助用户管理个人信息和查看个人数据。

**(2) 实现过程**

个人信息管理允许用户查看和编辑个人资料，包括头像、联系方式、健身目标等信息。用户修改信息后，前端将新数据发送到后端，后端验证并更新数据库。

健康档案管理允许用户记录和管理个人健康数据，包括身高、体重、当前体重，以及每日的健康记录。用户可以设置健身目标，系统根据这些信息为用户提供个性化的膳食推荐。

发布内容管理允许用户查看自己发布的帖子和食谱列表，方便用户管理自己的创作内容。

收藏管理允许用户查看和管理收藏的食谱和帖子，用户可以对不感兴趣的内容取消收藏。

### 4.3.4 食谱模块

**(1) 功能描述**

食谱模块实现食谱浏览、搜索、详情查看和推荐等功能，为用户提供丰富的食谱资源和个性化的推荐服务。

**(2) 实现过程**

食谱列表页面加载时，前端向后端请求食谱列表数据，后端从数据库查询食谱信息并返回给前端。前端使用分页组件展示食谱列表，支持用户翻页查看更多食谱。

食谱搜索功能允许用户输入关键词搜索食谱，后端根据关键词在食谱名称、描述等字段中进行模糊查询，返回匹配的食谱列表。

食谱详情页面展示完整的食谱信息，包括食谱名称、封面图片、健身目标、营养标签、食材列表、烹饪步骤、营养成分等详细信息。

食谱推荐功能根据用户的健身目标、健康档案和历史行为，通过推荐算法生成个性化的食谱推荐列表。

### 4.3.5 社区模块

**(1) 功能描述**

社区模块实现帖子发布、评论、审核、点赞、收藏和消息通知等功能，为用户提供交流分享的平台，促进用户之间的互动，并确保社区内容的安全性和质量。

**(2) 实现过程**

帖子发布功能允许用户创建新帖子，分享膳食经验和心得。用户输入帖子标题、内容等信息后，前端将数据发送到后端，后端保存帖子到数据库。系统设置帖子状态为"待审核"，管理员审核通过后，帖子状态变为"已通过"，用户会收到审核结果的消息通知。

帖子列表页面展示所有已通过审核的帖子，支持按时间排序和搜索过滤。用户可以点击帖子查看详情，参与评论和互动。

评论功能允许用户对帖子和食谱进行评论，表达观点和交流心得。后端保存评论内容，评论状态同样需要管理员审核通过后才会在前台展示。用户提交评论后，系统等待管理员审核，审核结果会通过消息通知用户。

点赞和收藏功能允许用户对喜欢的帖子和食谱表示支持。当用户点赞或收藏其他用户的帖子时，系统会向帖子作者发送互动通知，告知有人对其内容进行了点赞或收藏操作。

消息通知功能为用户提供系统消息和互动消息的接收与查看。系统消息包括帖子和评论的审核结果通知，如"您的帖子已通过审核"或"您的评论未通过审核"等。互动消息包括其他用户对其帖子的点赞和收藏通知，如"用户XXX点赞了您的帖子"等。用户可以在个人中心查看消息列表，了解未读消息数量，并标记消息为已读。

### 4.3.6 AI 助手模块

**(1) 功能描述**

AI 助手模块为用户提供智能问答服务，解答关于膳食营养、健康饮食等方面的问题，帮助用户获取专业的营养建议。

**(2) 实现过程**

用户向 AI 助手提问时，前端将问题发送到后端。后端调用智谱 AI 的 API 接口，将问题转发给 AI 模型处理，获取回答后返回给前端展示。

AI 助手可以回答用户关于膳食搭配、营养成分、健康饮食等方面的问题，提供个性化的建议和指导。

### 4.3.7 管理模块

**(1) 功能描述**

管理模块实现用户管理、食谱管理、食材管理、帖子管理、评论管理和统计报表等功能，帮助管理员管理系统的日常运营。

**(2) 实现过程**

用户管理功能允许管理员查看用户列表、搜索用户、查看用户详情等操作。管理员可以了解用户的注册信息和健康档案。

食谱管理功能允许管理员查看食谱列表、搜索食谱等操作。管理员可以浏览系统中所有用户提交的食谱信息。

食材管理功能允许管理员查看食材列表、搜索食材、查看食材详情等操作。管理员可以了解系统中所有食材的营养信息。

帖子管理功能允许管理员查看帖子列表、搜索帖子、审核帖子和删除违规帖子等操作。管理员可以对待审核的帖子进行审核操作，审核通过后帖子才会在前台展示。

评论管理功能允许管理员查看评论列表、搜索评论、审核评论和删除不当评论等操作。管理员可以对待审核的评论进行审核操作，审核通过后评论才会在前台展示。

统计报表功能为管理员提供系统数据统计，包括用户数量、用户性别分布、用户年龄分布、用户健身目标分布等数据的统计和可视化展示。管理员可以通过统计数据了解系统的运行情况，为系统优化提供数据支持。

### 4.3.8 推荐算法模块（没写）

**(1) 功能描述**

推荐算法是系统的核心功能之一，通过分析用户的健身目标、健康档案和历史行为，为用户推荐个性化的食谱。

**(2) 实现过程**

系统采用混合推荐算法，结合协同过滤和基于内容的推荐策略。推荐算法主要步骤包括用户特征提取、食谱特征提取、相似度计算和推荐生成。

用户特征提取从用户档案中获取健身目标、饮食禁忌等信息，从用户行为数据中提取偏好信息，计算用户对不同食谱的偏好权重。

食谱特征提取从食谱中获取健身目标、营养标签、饮食禁忌等信息，计算食谱的热度分数。

相似度计算包括协同过滤相似度、健身目标匹配度和营养标签匹配度。协同过滤通过计算用户之间的行为相似度，找到相似用户喜欢的食谱。

推荐生成根据用户类型分配不同权重，过滤不符合条件的食谱，按推荐分数排序生成推荐列表。

### 4.3.9 后端数据存储

**(1) 数据库设计**

系统使用 MySQL 数据库存储数据，主要数据表包括用户表（yonghu）、管理员表（users）、食谱信息表（shipuxinxi）、食材信息表（shicaixinxi）、帖子表（forum）、评论表、收藏表（storeup）、用户档案表（user_profile）、用户行为表（user_behavior）、食谱推荐表（recipe_recommendation）、消息表（message）等。

**(2) 数据访问层实现**

系统通过 MyBatis-Plus 简化数据库操作，数据访问层定义 Mapper 接口继承 BaseMapper，通过泛型指定操作的实体类型。MyBatis-Plus 提供了强大的 CRUD 功能，支持分页查询、条件构造等常用操作。

服务层实现业务逻辑，调用数据访问层的方法完成数据操作。控制器层接收前端请求，调用服务层方法处理业务，将结果返回给前端。

## 4.4 本章小结

本章详细介绍了智慧膳食分享与个性化推荐系统的实现过程，包括系统功能概述、部分界面展示、图表可视化实现和关键模块实现等方面。系统采用前后端分离的开发模式，前端使用 Vue 3 框架和 Element Plus 组件库，后端使用 Spring Boot 框架和 MyBatis-Plus 数据访问框架。系统实现了用户认证、个人中心、食谱管理、社区互动、AI 助手和管理后台等核心功能模块，通过混合推荐算法为用户提供个性化的膳食推荐服务，通过审核机制和消息通知功能保障社区内容的安全性和互动性，满足了健身人群的膳食管理需求。
