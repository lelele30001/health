# 基于SpringBoot的面向健身人群的智慧膳食分享与个性化推荐系统详细分析

## 1. 系统功能详细分析

### 1.1 登录与注册流程

#### 1.1.1 登录流程

**操作步骤**：
1. 用户打开系统登录页面 (`d:\file\college\0-food\code\project\manage_code\src\views\login.vue`)
2. 系统加载登录界面，显示账号、密码输入框和用户类型选择下拉框
3. 用户输入账号和密码
4. 用户选择用户类型（管理员/普通用户）
5. 用户点击「登录」按钮
6. 系统前端验证输入：
   - 检查账号是否为空
   - 检查密码是否为空
   - 检查用户类型是否选择（如果有多个用户类型）
7. 系统发送登录请求到后端：
   - 请求URL：`${tableName}/login?username=${loginForm.value.username}&password=${loginForm.value.password}`
   - 请求方法：POST
8. 后端验证账号密码：
   - 验证成功：返回token和用户信息
   - 验证失败：返回错误信息
9. 前端处理登录结果：
   - 登录成功：
     - 保存账号密码到缓存（如果选择「记住密码」）
     - 保存token到本地存储
     - 保存角色信息到本地存储
     - 保存用户表名到本地存储
     - 保存管理员名称到本地存储
     - 调用Vuex action获取用户信息并保存
     - 跳转到系统首页
   - 登录失败：
     - 显示错误提示信息

**技术实现**：
- 前端：Vue 3 + Element Plus 实现登录表单
- 后端：Spring Boot 实现登录验证逻辑
- 数据存储：MySQL 存储用户信息
- 认证方式：JWT token

#### 1.1.2 注册流程

**操作步骤**：
1. 用户打开系统注册页面 (`d:\file\college\0-food\code\project\client_code\src\views\pages\yonghu\register.vue`)
2. 系统加载注册界面，显示注册表单
3. 用户填写注册信息：
   - 用户账号
   - 用户密码
   - 确认用户密码
   - 用户姓名
   - 性别
   - 手机号码
   - 邮箱
4. 用户点击「注册」按钮
5. 系统前端验证输入：
   - 检查用户账号是否为空
   - 检查用户密码是否为空
   - 检查两次密码输入是否一致
   - 检查用户姓名是否为空
   - 检查手机号码格式是否正确（如果填写）
   - 检查邮箱格式是否正确（如果填写）
6. 系统自动生成DiceBear头像URL：
   ```javascript
   registerForm.value.touxiang = `https://api.dicebear.com/7.x/avataaars/svg?seed=${registerForm.value.yonghuzhanghao}`;
   ```
7. 系统发送注册请求到后端：
   - 请求URL：`yonghu/register`
   - 请求方法：POST
   - 请求数据：注册表单数据
8. 后端处理注册请求：
   - 验证注册信息
   - 创建用户账号
   - 保存用户信息到数据库
9. 前端处理注册结果：
   - 注册成功：显示成功提示，跳转到登录页面
   - 注册失败：显示错误提示信息

**技术实现**：
- 前端：Vue 3 + Element Plus 实现注册表单
- 后端：Spring Boot 实现注册逻辑
- 数据存储：MySQL 存储用户信息

## 2. 系统技术实现详细分析

### 2.1 前端技术

**核心技术栈**：
- **框架**：Vue 3
- **UI组件库**：Element Plus
- **图表库**：ECharts
- **状态管理**：Vuex
- **路由管理**：Vue Router
- **HTTP请求**：Axios

### 2.2 后端技术

**核心技术栈**：
- **框架**：Spring Boot
- **数据库**：MySQL
- **API设计**：RESTful API
- **认证授权**：JWT

**关键实现**：
- **登录验证**：实现JWT token生成和验证

- **个性化推荐**：实现基于用户健身目标的推荐算法

## 4. 系统安全性详细分析

### 4.1 前端安全

**实现措施**：
- **输入验证**：对用户输入进行验证，防止XSS攻击
- **CSRF防护**：实现CSRF防护，防止跨站请求伪造
- **敏感信息保护**：不在前端存储敏感信息
- **token管理**：使用localStorage存储token，实现无状态认证

### 4.2 后端安全

**实现措施**：
- **认证授权**：使用JWT实现认证授权，确保API安全
- **密码加密**：对用户密码进行加密存储，防止密码泄露
- **SQL注入防护**：使用参数化查询，防止SQL注入
- **API限流**：实现API限流，防止恶意请求
- **权限控制**：实现基于角色的权限控制

### 4.3 服务器安全

**实现措施**：
- **防火墙配置**：配置防火墙，限制访问
- **定期更新**：定期更新系统和依赖，修复安全漏洞
- **日志监控**：实现日志监控，及时发现异常
