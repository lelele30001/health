# 第二章 关键技术概述
本章对智慧膳食分享与个性化推荐系统开发过程中使用到的关键技术进行介绍，分析其特点与应用场景，为本系统的开发提供技术支撑。系统采用前后端分离架构，结合多种现代技术栈实现功能需求。

在当今的互联网时代，有两种常见的软件架构框架，即C/S架构和B/S架构。 前者最初由Borland公司开发，后者是由微软公司推动其发展。  “C/S”是“Client/Server”的缩写，即“客户端/服务器”架构[29]。C/S架构 通过为客户端和服务器分配不同的任务，有效地减少了两者之间通信的成本。但 C/S 架构对客户端和服务器的硬件设备具有一定的要求。根据不同的应用软件需 求，客户端和服务器均需安装和配置特定的软件环境。在软件升级过程中，为保 证软件的兼容性与性能，客户端也需要进行同步更新与安装。随着用户数量的增 加，C/S架构出现硬件成本、升级维护以及运行效率等一系列问题，导致无法满 足部分用户的需求。  “B/S”是“Broweser/Server”的缩写，即“浏览器/服务器”架构。B/S 架 构对C/S架构进行了升级和优化，其中的业务操作是通过浏览器与服务器之间的 交互完成的。B/S架构图如图2-1所示：

## 2.1 Spring Boot 框架
Spring Boot 是由 Pivotal 团队开发的开源框架，旨在简化 Spring 应用的初始化和开发过程。本系统采用 Spring Boot 2.2.2 版本作为后端基础框架，利用其"约定大于配置"的设计理念，实现了自动装配和快速开发。

Spring Boot 的核心优势体现在以下方面：
(1) 内嵌 Tomcat 等服务器，无需额外部署
(2) 自动配置依赖和环境，减少手动配置工作量
(3) 提供丰富的 starter 依赖，简化项目搭建
(4) 与其他框架良好集成，如 MyBatis、Shiro 等
(5) 支持生产环境监控和管理功能

这些特性使得开发人员能够专注于业务逻辑实现，提高开发效率，同时保证系统的稳定性和可维护性。（使用文件：d:\file\college\0-food\code\project\server_code\pom.xml）

## 2.2 MyBatis 与 MyBatis-Plus
MyBatis 是一款优秀的持久层框架，支持自定义 SQL、存储过程和高级映射。本系统使用 MyBatis 作为 ORM 框架，结合 MyBatis-Plus 2.3 版本增强功能。

MyBatis-Plus 作为 MyBatis 的增强工具，提供的核心功能包括：
(1) 自动生成 CRUD 方法，减少重复代码
(2) 支持 Lambda 表达式，简化查询条件构建
(3) 提供分页插件，方便数据分页处理
(4) 支持乐观锁和逻辑删除等实用功能

这些特性大大简化了数据库操作代码，提高了开发效率，同时保持了 MyBatis 的灵活性。（使用文件：d:\file\college\0-food\code\project\server_code\pom.xml）

## 2.3 MySQL 数据库
MySQL 是一种关系型数据库管理系统，以其开源、稳定、高效的特点被广泛应用于各类应用系统。本系统采用 MySQL 作为数据存储方案，用于存储用户信息、膳食数据、帖子内容等核心数据。

MySQL 的主要优势包括：

(4) 丰富的索引类型，提高查询效率
(5) 完善的社区支持和文档

通过合理的数据库设计和索引优化，系统能够高效处理用户请求和数据操作。（使用文件：d:\file\college\0-food\code\project\server_code\src\main\resources\application.yml）

## 2.4 Apache Shiro 安全框架
Apache Shiro 是一款功能强大且易于使用的 Java 安全框架，本系统使用 Shiro 1.3.2 版本实现用户认证和权限管理。

Shiro 提供的核心功能包括：
(1) 身份认证：验证用户身份，确保合法用户访问
(2) 权限授权：控制用户对资源的访问权限
(3) 会话管理：维护用户会话状态
(4) 密码加密：保护用户密码安全
(5) 缓存支持：提高认证和授权效率

通过 Shiro 的集成，系统实现了安全的用户登录、权限控制和数据保护，确保系统的安全性。（使用文件：d:\file\college\0-food\code\project\server_code\pom.xml）

## 2.5 Vue 3 框架
Vue 3 是一款流行的前端 JavaScript 框架，以其轻量、响应式、易上手的特点被广泛应用。本系统采用 Vue 3.2.13 版本作为前端框架，利用其 Composition API 实现组件化开发。

Vue 3 的核心特性包括：
(1) 响应式系统：基于 Proxy 实现的数据响应式
(2) Composition API：提供更灵活的代码组织方式
(3) 虚拟 DOM：提高渲染性能
(4) 组件化开发：支持组件复用和维护
(5) 生态系统丰富：拥有众多第三方库和工具

通过 Vue 3 的 Composition API，系统实现了模块化的前端代码结构，提高了组件的复用性和可维护性。（使用文件：d:\file\college\0-food\code\project\manage_code\package.json）
