# 毕业设计中期报告

## 一、项目概述

本项目是一个基于Spring Boot和Vue.js的美食推荐系统，旨在为用户提供个性化的食谱和帖子推荐服务，同时集成健康管理功能。系统采用前后端分离架构，后端使用Spring Boot框架，前端使用Vue.js框架，数据库使用MySQL。

## 二、已完成的研究内容及工作任务

### 2.1 系统架构搭建

- **后端架构**：完成了Spring Boot框架的搭建，包括项目初始化、依赖配置、数据库连接等。
- **前端架构**：完成了Vue.js项目的搭建，包括路由配置、组件结构设计、状态管理等。
- **数据库设计**：完成了数据库表结构的设计，包括用户表、食谱表、帖子表、用户行为表、健康记录表等。

### 2.2 核心功能实现

- **用户管理模块**：实现了用户注册、登录、个人信息管理等功能。
- **食谱管理模块**：实现了食谱的增删改查、食材管理等功能。
- **帖子管理模块**：实现了帖子的发布、查看、评论等功能。
- **用户行为记录**：实现了用户对帖子的点赞、收藏、踩等行为的记录功能。
- **健康管理模块**：实现了用户健康数据的记录、分析和管理功能，包括体重、BMI、每日热量摄入等数据的记录和分析。
- **后台管理员模块**：实现了管理员登录、用户信息管理、帖子管理、健康数据管理等功能，管理员可以查看每个用户的信息，查看每一条帖子，并对这些信息进行增删改查。
- **京东食材购买**：实现了跳转到京东购买食材的功能，用户可以通过系统直接跳转到京东购买相关食材。
- **AI问答功能**：设计了AI问答界面，计划对接阿里云的DeepSeek API，实现用户与AI的交互功能。

### 2.3 个性化推荐系统实现

- **推荐架构**：实现了协同过滤 + 内容推荐的双层推荐结构。
- **协同过滤推荐**：基于用户收藏行为，找出与当前用户收藏帖子重合度最高的相似用户，提取相似用户收藏过但当前用户未收藏的帖子，按相似度排序后优先推荐。
- **内容推荐**：基于用户健身目标、饮食禁忌等信息，推荐最适合的帖子，作为协同过滤推荐的兜底方案。
- **冷启动解决方案**：对于新用户或无收藏行为的用户，直接使用内容推荐，根据用户资料推荐最适合的帖子。
- **推荐优化策略**：在内容推荐中加入随机采样，避免过度热门偏见；过滤用户已收藏、已点赞的帖子，避免重复推荐。

### 2.4 系统测试与验证

- **功能测试**：对系统的核心功能进行了测试，包括用户注册登录、食谱管理、帖子管理、健康数据记录、推荐系统等功能，确保系统功能正常运行。
- **逻辑测试**：验证了系统的业务逻辑是否正确，包括推荐算法的准确性、健康数据的计算、权限管理的有效性等。
- **性能测试**：测试了系统的响应速度和并发处理能力，确保系统在正常使用情况下能够稳定运行。
- **兼容性测试**：测试了系统在不同浏览器和设备上的显示效果，确保响应式设计的有效性。

## 三、技术实现细节

### 3.1 后端实现

- **技术栈**：Spring Boot、MyBatis、Spring Security等。
- **API设计**：采用RESTful风格的API设计，实现了用户、食谱、帖子、推荐、管理员、健康管理等相关接口。
- **数据处理**：使用MyBatis进行数据库操作，实现了高效的数据查询和更新。
- **安全认证**：使用Spring Security实现了用户认证和授权，区分普通用户和管理员权限。
- **健康数据管理**：实现了用户健康数据的记录、分析和管理功能，包括体重、BMI、每日热量摄入等数据的计算和分析。

### 3.2 前端实现

- **技术栈**：Vue.js、Vue Router、Vuex、Element UI等。
- **组件设计**：设计了用户、食谱、帖子、推荐、管理员、健康管理等相关组件。
- **状态管理**：使用Vuex进行全局状态管理，实现了数据的共享和同步。
- **响应式设计**：实现了响应式布局，适配不同设备的显示。
- **健康管理界面**：设计了健康数据记录、分析和管理界面，包括体重、BMI、每日热量摄入等数据的展示和分析。
- **管理员界面**：设计了管理员专用界面，实现了用户信息、帖子和健康数据的管理功能。
- **京东跳转按钮**：在食谱详情页面添加了京东购买食材的跳转按钮，直接使用京东商品链接，用户点击后跳转到京东购买相关食材。
- **AI问答界面**：设计了AI问答界面，计划对接阿里云的DeepSeek API，实现用户与AI的交互功能，获取关于美食、食谱、健康等相关问题的回答。

### 3.3 推荐系统实现

- **协同过滤算法**：使用SQL实现了基于用户的协同过滤算法，计算用户相似度，推荐相似用户收藏的帖子。
- **内容推荐算法**：基于用户资料和帖子内容的匹配度，推荐最适合的帖子。
- **混合推荐策略**：优先使用协同过滤推荐，如果不足3条则使用内容推荐补充。
